= Goedgepickt Orders Export

== Introduction
This tool allows the user to export orders from https://www.goedgepickt.nl/ in a CSV format that can be imported into https://www.routigo.com/

== Technical Design

=== Stack
The application is written in Go + Fyne.

The application is developed on Macos, but the target is Windows. Github Actions is used to compile to Windows.
No installer is required.

== Development

Run the application locally:

[source,shell]
----
zsh -lic 'go run ./cmd/goedgepickt-orders-export'
----

Build the application locally:

[source,shell]
----
zsh -lic 'go build ./cmd/goedgepickt-orders-export'
----

The Codex app shell in this workspace does not currently expose `go` on the non-login PATH, even though Go is installed through `mise`. Using `zsh -lic '...'` ensures the login shell loads the `mise` environment first.

== Windows Build

GitHub Actions builds the first `windows/amd64` target on pushes to `main` and on pull requests, and uploads the resulting executable as a workflow artifact named `goedgepickt-orders-export-windows-amd64`.

When you push a version tag such as `v1.0.0`, the same workflow also creates a GitHub Release and attaches the Windows executable there, which is the easiest place to download it from the GitHub website.

----
git tag v1.0.0
git push origin v1.0.0
----

The module `go` version is currently kept at `1.24.6` so it remains compatible with the Go toolchain bundled inside the current `fyne-cross` Windows image, while local development can still use a newer Go release.

== Vendor Information
=== Goedgepickt API
API documentation: https://www.goedgepickt.dev/

Assuming you get an API key and store this in file `token.txt`, the command below illustrates the API.

.Get Orders
----
curl -H "Authorization: Bearer $(cat token.txt)" \
  --location 'https://account.goedgepickt.nl/api/v1/orders?searchAttribute=status&searchDelimiter=%3D&searchValue=ready_for_picking&page=1&orderBy=createDate&orderByDirection=asc&perPage=50' > orders.json
----

Ths API returns a paginated result. Default 25 items per page, max 50.
